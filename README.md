# Elder Guardian AI

AI-powered elder financial exploitation early-warning system.

A family member uploads a CSV of an elderly person's bank transactions. A LangGraph multi-agent AI pipeline analyzes spending patterns, scores exploitation risk from 0-100, and if the score exceeds 65, sends an email alert to a trusted emergency contact with a plain-English explanation generated by GPT-4o-mini.

## How It Works

1. Family member signs up and adds an emergency contact (name, phone, email)
2. Uploads a CSV of the senior's bank transactions
3. A 5-agent LangGraph pipeline runs automatically:
   - **Agent 1 - Ingestion:** Validates and cleans the CSV
   - **Agent 2 - Feature Engineering:** Computes 8 features per transaction (velocity, payee patterns, time risk, transfer detection)
   - **Agent 3 - Anomaly Detection:** Ensemble of Isolation Forest + LSTM Autoencoder + graph heuristics + SHAP explainability
   - **Agent 4 - Narrative:** GPT-4o-mini writes a warm, non-alarmist explanation for the family
   - **Agent 5 - Dispatch:** Sends email alert to the emergency contact
4. Dashboard shows risk timeline chart, upload summary, and alert log

## Stack

- **Frontend:** Next.js 14 (App Router), TypeScript, Tailwind CSS, shadcn/ui, Recharts
- **Backend:** Python FastAPI
- **Pipeline:** LangGraph multi-agent (5 agents)
- **ML:** Isolation Forest, LSTM Autoencoder, graph heuristics, SHAP
- **LLM:** OpenAI GPT-4o-mini (narrative generation)
- **Database:** Supabase (PostgreSQL + Auth + Row Level Security)
- **Alerts:** Resend (email)


## Demo Scenarios

Three synthetic CSV files are included for demos:

| File | Risk Score | Description |
|------|-----------|-------------|
| normal_baseline.csv | ~50 | 90 days of normal senior spending — no alert |
| grooming_then_drain.csv | ~92 | Small gifts building trust over 60 days, then large drains — alert fires |
| sudden_exploitation.csv | ~77 | Sudden large wire/crypto transfers — alert fires |

## Setup

### Prerequisites
- Python 3.11
- Node.js 18+
- Supabase account
- OpenAI API key
- Resend account

### Backend
```bash\ncd backend\npy -3.11 -m venv .venv\n.venv\\Scripts\\Activate.ps1  # Windows\npip install -r requirements.txt\n# Copy .env.example to .env and fill in your keys\npython -m uvicorn backend.main:app --reload --port 8000\n```\n
### Frontend
```bash\ncd frontend\nnpm install\n# Copy .env.local.example to .env.local and fill in your keys\nnpm run dev\n```\n
## Environment Variables

### Backend (.env)
```\nSUPABASE_URL=\nSUPABASE_ANON_KEY=\nSUPABASE_SERVICE_ROLE_KEY=\nOPENAI_API_KEY=\nRESEND_API_KEY=\nRESEND_FROM_EMAIL=\n```\n
### Frontend (.env.local)
```\nNEXT_PUBLIC_SUPABASE_URL=\nNEXT_PUBLIC_SUPABASE_ANON_KEY=\nNEXT_PUBLIC_API_URL=http://localhost:8000\n```\n
